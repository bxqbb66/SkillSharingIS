═══════════════════════════════════════════════════════════
  一、开场 + 界面总览（1分钟）
═══════════════════════════════════════════════════════════

【画面：网站首页，电脑浏览器全屏展示】

大家好，这是"吉大互助"——吉林大学校园技能共享平台，
基于 React 18 + Vite + Tailwind CSS 构建，
采用 Mobile First 响应式设计。

在进入详细业务流程之前，先快速浏览一下系统有哪些页面。

【操作：边点击各个Tab边介绍，每个页面停留3-5秒】

学生端一共 10 个页面：

  首页广场——技能和需求的双Tab浏览，支持搜索和分类筛选
  发布页——双表单，可以发布技能或发布需求
  消息页——聊天列表和系统通知
  我的工单——按状态筛选查看所有工单
  个人中心——信用分、钱包、评价、快捷入口
  技能/需求详情页——查看完整信息，预约或接单
  工单详情页——进度追踪、操作按钮、资金状态
  他人主页——查看任意用户的公开信息和评价
  申诉记录——查看申诉历史和裁决结果
  登录页——统一身份认证入口

管理端有 7 个页面：

  数据看板——平台指标一览
  内容审核——审核技能和需求的发布
  工单监管——查看和标记所有工单
  用户管理——用户列表和信用分调整
  申诉仲裁——处理用户申诉
  交易记录——查看所有资金流水
  规则配置——分类、价格区间、公告管理

整个系统的视觉统一采用吉林大学校徽蓝 #003366 为主色调，
所有页面都支持手机端和电脑端的响应式切换。

好，界面看完了。下面我带着一个完整的业务案例，
把上面的页面串起来，演示整个流程。

【操作：选择一个待验收的工单演示】

  填写裁决描述

  设定双方信用分变动
  确认裁决

裁决后工单关闭，资金按比例退款，信用分自动调整。

  八、设计亮点特写（1分钟）
═══════════════════════════════════════════════════════════

【逐项演示，可配合缩放窗口和慢动作】

1. 响应式布局（10秒）
   - 拖窄浏览器 → 自动切换底部Tab导航，单列布局
   - 拉宽 → 左侧固定侧边栏 + 2-3列网格
   - Mobile First 原则，md: 前缀覆写电脑端

2. 品牌与色彩（5秒）
   - 主色吉大蓝 #003366，全站统一
   - 按钮、Tab、链接、状态标识全部使用主色调
   - 首页展示吉林大学校徽

3. 卡片设计（10秒）
   - 四色渐变色头部：学业蓝📚 / 技术紫💻 / 生活绿🏠 / 文体橙🎯
   - 头部叠加技能标签文字
   - 悬停上移 + 阴影加深（0.3s过渡）

4. 骨架屏与空状态（10秒）
   - 首页加载时卡片骨架屏 + 闪烁动画
   - 详情页骨架屏占位
   - 空状态：搜索无结果、无工单、无评价都有专门的SVG图标和引导文案

5. 头像系统（5秒）
   - DiceBear API 根据学号自动生成差异化卡通头像
   - 鲜花饼使用自定义上传头像
   - 所有头像可点击跳转到该用户主页

6. 消息通知（10秒）
   - 聊天列表 + 系统通知双Tab
   - 系统通知带彩色类型标签
   - 聊天气泡式对话，回车发送

7. 信用体系（5秒）
   - 信用分 0-1000 + 等级 A/B/C/D 双轨
   - 等级徽章：A绿 / B蓝 / C黄 / D红
   - 评价自动影响，管理员可手动调整

8. 状态可视化（5秒）
   - 工单状态横幅颜色随状态变化：黄→蓝→紫→绿→红
   - 进度时间轴圆点+连线
   - 资金状态实时显示
   - 申诉/仲裁状态全程追踪

═══════════════════════════════════════════════════════════
  九、结尾（15秒）

  发布 → 审核 → 上架 → 下单 → 工单流转 → 验收结算 → 评价 → 信用更新

覆盖学生端 10 个页面 + 管理端 7 个页面，
