  一、开场（30秒）

【画面：网站首页，电脑浏览器全屏展示】

大家好，这是"吉大互助"——吉林大学校园技能共享平台。
它是一个连接校园内"有技能的人"和"有需求的人"的互助系统，
学生可以发布自己的技能变现，也可以发布需求悬赏求助。

今天我带大家走一遍完整的业务流程：
  鲜花饼发布技能 → 管理员审核 → 王雨晴接单 → 工单完成 → 评价闭环

整个系统基于 React 18 + Vite + Tailwind CSS 构建，
采用 Mobile First 响应式设计，手机端和电脑端都适配。

现在进入了工单的核心页面。当前状态是"待确认"，

中间是进度时间轴，用圆点和连线展示每一步，

往下是任务信息：任务描述、约定交付时间、交付要求，

进度时间轴完整展示了：

工单完成后，出现"评价对方"按钮。点击后弹出评价窗口：

  5 星评分（点击星星）

═══════════════════════════════════════════════════════════
  七、申诉流程（可选，30秒）
═══════════════════════════════════════════════════════════

【操作：选择一个待验收的工单演示申诉】

如果验收出现争议，需求方可以"发起申诉"。
在工单详情页点击"发起申诉"，
填写申诉原因，提交后工单进入"申诉中"状态。

管理员在后台"申诉仲裁"页面可以看到申诉列表，
点击进入裁决面板：

  填写裁决描述
  设置退款比例（比如 50%）
  设定双方信用分变动（服务方 -5，需求方 +3）
  确认裁决

裁决完成后，工单关闭，资金按比例退款，
双方信用分自动调整。

═══════════════════════════════════════════════════════════
  八、设计亮点特写（1分钟）
═══════════════════════════════════════════════════════════

【逐项演示，可快进或截图展示】

1. 响应式布局
   - 拖窄浏览器窗口 → 手机端自动切换为底部Tab导航
   - 拉宽窗口 → 电脑端左侧固定侧边栏 + 多列网格
   - 采用 Mobile First 原则，所有样式先在手机端定义，md: 覆写电脑端

2. 品牌视觉
   - 主色为吉林大学校徽蓝 #003366
   - 所有按钮、Tab高亮、链接统一使用主色调
   - 首页顶部展示吉林大学校徽

3. 卡片头部设计
   - 学业辅导 → 蓝色渐变 + 📚 emoji
   - 技术服务 → 紫色渐变 + 💻 emoji
   - 生活服务 → 绿色渐变 + 🏠 emoji
   - 文体服务 → 橙色渐变 + 🎯 emoji
   - 卡片悬停时上移 + 阴影加深（0.3s 过渡动画）

4. 骨架屏加载
   - 首页首次加载时显示卡片骨架屏
   - 灰白色闪烁动画模拟内容加载过程
   - 详情页也有骨架屏占位

5. 空状态设计
   - 搜索无结果时展示 SVG 搜索图标 + 引导文字
   - 无工单时展示文档图标 + "去广场看看吧"
   - 每个空页面都有专门的插画和文案

6. 头像系统
   - 使用 DiceBear API 根据学号自动生成卡通头像
   - 鲜花饼使用自定义上传的头像

7. 通知与消息
   - 消息页面双Tab（聊天 + 系统通知）
   - 系统通知带彩色类型标签：工单通知/审核通知/申诉通知/评价通知
   - 聊天支持实时发送，聊天气泡式对话

8. 信用体系
   - 信用分 + 信用等级（A/B/C/D）双轨展示
   - 等级徽章颜色对应（绿/蓝/黄/红）
   - 评价影响信用分，管理员可手动调整

【画面：回到首页，缓慢缩小浏览器窗口展示响应式切换】

总结一下，吉大互助平台实现了完整的校园技能交易闭环：
  发布内容 → 管理员审核 → 广场上架 → 下单接单
  → 工单状态流转 → 验收结算 → 双向评价 → 信用更新

整个流程覆盖了学生端的 10 个页面和管理端的 7 个页面，
支持手机和电脑双端访问。

技术栈：React 18 + Vite + Tailwind CSS + React Router v6
项目开源地址见 GitHub。
