# 吉大互助平台 — 界面设计说明与演示指南

---

## 一、界面设计总览

### 1.1 设计理念

本平台采用 **Mobile First** 响应式设计策略，以吉林大学校徽蓝 `#003366` 为主色调，打造简洁、专业、可信赖的校园互助体验。

- **手机端**：底部 Tab 导航 + 单列卡片流，仿微信小程序风格
- **电脑端**：左侧固定侧边栏 + 多列网格布局，仿闲鱼网页版风格

### 1.2 全局主题

| 设计要素 | 取值 |
|----------|------|
| 主色 | `#003366`（吉大蓝） |
| 辅色 | `#A4212E`（校徽红，用于紧急标记） |
| 背景 | `#F0F2F5` |
| 字体 | 系统默认无衬线字体 |
| 圆角 | 卡片 12px，按钮 8-12px |
| 阴影 | 卡片 `shadow-sm`，悬停 `shadow-md` |

## 二、学生端页面说明

### 2.1 登录/注册页 (`/login`)

> **[ 插入截图：登录页手机端截图 ]**

**功能描述**：统一身份认证入口。学生输入学号和密码即可登录，首次登录自动创建账号。管理员使用 `admin / admin123` 自动跳转管理后台。

**设计要点**：居中卡片布局，顶部展示吉林大学校徽，Tab 切换登录/注册，表单简洁，底部有测试账号提示。

---

### 2.2 首页广场 (`/home`)

> **[ 插入截图：首页技能广场手机端截图 ]**
> **[ 插入截图：首页需求广场电脑端截图 ]**

**功能描述**：平台核心页面，展示所有已审核通过的技能和需求。顶部导航栏含校徽、标题、用户头像入口。搜索框支持关键词筛选。技能/需求双 Tab 切换。分类筛选胶囊按钮（全部/学业/技术/生活/文体）。卡片以渐变色头部 + 分类 emoji + 标签文字呈现，悬停有上浮动效。

**卡片信息**：发布者头像（DiceBear）+ 姓名 + 信用等级 + 分类标签 + 描述 + 价格/预算 + 操作按钮

**空状态**：搜索无结果时展示 SVG 图标 + 引导文字。

---

### 2.3 发布页 (`/publish`)

> **[ 插入截图：发布技能表单截图 ]**
> **[ 插入截图：发布需求表单截图 ]**

**功能描述**：双 Tab 表单（发布技能 / 发布需求），字段严格对应数据库设计。必填项有红色星号标记，前端校验。提交后显示"发布成功，等待审核"弹窗。

**技能表单字段**：类别（下拉）、标签、价格、可服务时间、描述、资质（选填）、作品上传区

**需求表单字段**：类型（下拉）、标签、任务描述、预算、截止时间（日期选择器）、交付要求（选填）、紧急开关

---

### 2.4 技能/需求详情页 (`/skill/:id`, `/demand/:id`)

> **[ 插入截图：技能详情页截图 ]**

**功能描述**：左侧展示详情内容（标签、类别、价格、描述、可服务时间、作品展示、历史评价），右侧/底部展示发布者信息卡片（头像可点击跳转他人主页）和操作按钮（立即预约/我要接单）。确认后生成工单并跳转。

**设计要点**：手机端底部悬浮操作栏，电脑端右侧 Sticky 面板。

---

### 2.5 工单详情页 (`/order/:id`)

> **[ 插入截图：工单详情页 — 进行中状态 ]**
> **[ 插入截图：工单详情页 — 待验收状态 ]**

**功能描述**：完整工单生命周期管理。顶部状态横幅（颜色随状态变化），进度时间轴（圆点+连线），任务信息（描述、交付时间、资金状态），对方信息卡片，动态操作按钮区。

**状态流转**：
```
待确认 → [确认] → 进行中 → [提交成果] → 待验收 → [验收] → 已完成 → [评价]
                                                    ↘ [申诉] → 申诉中 → [裁决] → 已关闭
```

## 四、明天演示指南

### 4.1 准备清单

- [ ] 确保电脑安装了 Node.js
- [ ] 打开终端，进入项目目录
- [ ] 运行 `npm install`（如果之前没装过依赖）
- [ ] 运行 `npm run dev` 启动服务
- [ ] 打开浏览器 `http://localhost:5173`
- [ ] **重要：打开浏览器开发者工具 → Application → Local Storage → 清空 `jlu-skill-share-state` 和 `jlu-skill-share-current-user-id`**（清理旧缓存数据，确保干净启动）
- [ ] 准备一个手机模拟器或直接用浏览器手机模式（F12 → 切换设备工具栏）

### 4.2 演示路线（10-15 分钟）

#### 阶段一：学生端核心闭环（5 分钟）

1. **注册/登录** → 输入学号 `20210001`，密码任意 → 进入首页（鲜花饼账号）
2. **浏览广场** → 展示技能广场和需求广场的卡片、分类筛选、搜索功能
3. **发布技能** → 底部 Tab 点"发布" → 填写表单 → 提交 → 看到"等待审核"提示
4. **发布需求** → 切换到"发布需求" → 填写表单 → 提交
5. **切换手机/电脑视图** → 浏览器缩小窗口展示响应式布局切换

#### 阶段二：管理端审核（3 分钟）

6. **管理员登录** → 在 `/login` 输入 `admin / admin123` → 自动跳转管理后台
7. **内容审核** → 进入"内容审核" → 看到刚才发布的技能/需求 → 点击"通过"
8. **回学生端** → 侧边栏点"返回学生端" → 首页刷新 → 刚才发布的技能已出现在广场
9. **用户管理** → 回到管理端 → 用户管理 → 看到鲜花饼在列表中 → 展示信用分调整

#### 阶段三：工单闭环（4 分钟）

10. **下单** → 学生端广场点击一个技能 → 技能详情 → 点"立即预约" → 跳转工单详情
11. **确认工单** → 点"确认接单并开始服务" → 状态变为"进行中" → 资金冻结
12. **提交成果** → 点"提交交付成果" → 输入说明 → 提交 → 状态变为"待验收"
13. **验收** → 切换为需求方视角或用同一个工单 → 点"确认验收并付款" → 状态"已完成"
14. **评价** → 点"评价对方" → 打 5 星 → 提交评价 → 信用分自动更新

#### 阶段四：申诉流程（2 分钟）

15. **发起申诉** → 另一个工单 → 待验收状态 → 点"发起申诉" → 填写原因
16. **管理端裁决** → 管理端 → 申诉仲裁 → 看到新申诉 → 填写裁决 → 退款 50% → 扣服务方 10 分

### 4.3 演示小技巧

- **手机模式**：按 F12 → 点击设备工具栏图标（手机/平板图标）→ 选 iPhone 12 Pro 或 Pixel 5
- **清数据**：如果演示中数据乱了，清空 localStorage 刷新即可
- **快速切换账号**：清空 localStorage → 重新登录新学号
- **重点强调**：响应式布局（拉大拉小浏览器窗口）、颜色状态标识、骨架屏加载、卡片动效
