# 第七步开发任务：他人主页（公开信息展示）

我们已经有了一个完善的“我的”个人中心，现在需要开发**“他人主页”**，让用户在交易前可以了解对方。这与“我的个人中心”不同，只展示公开信息，不能修改。

请开发他人主页组件，具体需求如下：

1. **路由与入口**：
   - 新建路由 `/user/:id`。
   - **修改现有入口**：在“技能详情页”和“需求详情页”右侧/底部的“发布者信息”卡片上，将头像和姓名变成可点击的链接，点击后跳转到该发布者的他人主页。
   - 在“评价列表”中，评价人的头像/昵称也可点击跳转到该评价人的主页。

2. **页面布局（响应式）**：
   - 手机端：垂直单列滚动。
   - 电脑端：顶部展示用户信息横幅，下方展示具体内容。

3. **顶部用户信息横幅**（对应 user_information）：
   - 大头像 + 姓名 + 学院专业。
   - 信用分（credit_score）+ 信用等级徽章（credit_level，保持和之前一样的颜色逻辑）。
   - 统计数据概览：发布技能数、发布需求数、历史成交量（用 Mock 数据计算）。

4. **Tab 内容区**：
   - Tab 1：**Ta的技能**。展示该用户发布的技能卡片列表（复用首页的技能卡片样式），点击可跳转技能详情页。
   - Tab 2：**Ta的需求**。展示该用户发布的需求卡片列表，点击可跳转需求详情页。
   - Tab 3：**Ta的评价**（核心）。展示该用户收到的所有评价列表（评价人头像、星级 star_score、评价内容 evaluation_text、评价时间）。

5. **Mock 数据与逻辑**：
   - 请扩展 Mock 数据，至少生成 3 个不同的模拟用户（包括当前登录的“自己”，和另外 2 个“他人”）。
   - 不同用户拥有的技能、需求、评价必须不同。
   - 详情页跳转过来时，必须根据 URL 中的 `:id` 从 Mock 数据中找到对应的用户并渲染其信息。找不到则提示“用户不存在”。
