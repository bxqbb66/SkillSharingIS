# 第三步开发任务：技能/需求详情页面

请继续参考 CONTEXT.md 中的数据字典。首页和发布页已完成，现在我们需要开发**“详情页”**，让供需双方能够匹配并生成订单。

请创建两个详情页面组件：**技能详情页** 和 **需求详情页**。

具体需求如下：

1. **页面布局（响应式）**：
   - 手机端：从上到下依次展示信息，底部悬浮固定的操作按钮栏。
   - 电脑端：左侧展示详情内容（占 2/3 宽度），右侧展示发布者信息卡片和操作按钮（占 1/3 宽度，类似闲鱼/58同城的布局）。

2. **技能详情页**（面向需求方查看，对应 skill_information）：
   - 顶部：技能标签（skill_tag）、技能类别（skill_category）。
   - 核心信息：服务价格（service_price，醒目展示）、可服务时间（available_time）。
   - 详细说明：技能描述（skill_description）、资质信息（qualification_info）。
   - 附件展示：展示作品图片区域（使用占位图）。
   - 右侧/底部悬浮：发布者信息（头像、姓名、信用等级 credit_level），以及一个醒目的【立即预约】按钮。

3. **需求详情页**（面向服务方查看，对应 demand_information）：
   - 顶部：需求标签（demand_tag）、紧急标记（urgent_flag，红色高亮展示）。
   - 核心信息：预算金额（budget_amount，醒目展示）、截止时间。
   - 详细说明：任务描述（task_description）、交付要求（delivery_requirement）。
   - 右侧/底部悬浮：发布者信息（头像、姓名），以及一个醒目的【我要接单】按钮。

4. **交互逻辑（核心）**：
   - 在首页的卡片上，点击任意位置，跳转到对应的详情页（技能卡片跳技能详情，需求卡片跳需求详情）。路由建议为 `/skill/:id` 和 `/demand/:id`。
   - 点击【立即预约】或【我要接单】按钮后，弹出一个二次确认弹窗：“确认要接单/预约吗？”。
   - 点击确认后，提示“工单生成成功，请尽快联系对方”，并自动跳转到一个“我的工单/订单”页面（目前该页面可先设为空白，只显示“暂无进行中的工单”即可，后续再完善）。

5. **Mock 数据衔接**：
   - 请确保详情页能从首页的 Mock 数据中获取对应的 ID 并展示对应内容。如果目前没有路由传参，请先在详情页内写死一条详细的 Mock 数据展示。
