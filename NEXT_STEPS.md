# 第五步开发任务：消息页面与工单详情页

请继续参考 CONTEXT.md 中的数据字典。现在我们需要完善交易的核心闭环：沟通与履约。

请开发两个核心模块：**消息页面** 和 **工单详情页**。

## 一、 工单详情页（核心优先级，对应 order_record 和 evaluation_information）
这是交易履约的核心页面，强调“过程可见”。

1. **顶部状态区**：
   - 大字展示当前工单状态（order_status），不同状态用不同颜色背景（待确认=黄，进行中=蓝，待验收=紫，已完成=绿，申诉中=红）。
   - 显示报酬金额（reward_amount）。

2. **进度时间轴**：
   - 使用垂直时间轴组件展示工单进度日志（progress_log）。
   - 节点包括：工单创建 -> 服务方开始执行 -> 服务方提交成果 -> 需求方验收通过 -> 双方互评完成。
   - 请用 Mock 数据生成至少3个不同进度状态的工单。

3. **任务信息区**：
   - 任务描述（task_description）、约定交付时间、实际交付时间。

4. **动态操作按钮区（关键）**（根据工单状态和用户身份动态显示）：
   - 如果状态是“进行中”且我是服务方：显示【提交交付成果】按钮（点击后弹窗上传截图/输入说明，状态变为待验收）。
   - 如果状态是“待验收”且我是需求方：显示【确认验收并付款】按钮 + 【发起申诉】按钮。
   - 如果状态是“已完成”且未评价：显示【评价对方】按钮，点击弹出评价框（星级评分 star_score + 文字评价 evaluation_text）。

5. **路由与跳转**：
   - 路由设为 `/order/:id`。
   - 修改之前的“个人中心-我的工单”列表和“详情页-接单成功提示”，使其正确跳转到此工单详情页。

## 二、 消息页面
对接报告中 E6 消息通知服务，作为用户沟通的入口。

1. **布局**：顶部 Tab 切换“聊天”和“系统通知”。

2. **聊天列表**（对应 IM 占位）：
   - 展示最近的聊天对象（对方头像、姓名、最后一条消息摘要、时间）。
   - 点击后跳转到一个简单的聊天对话界面（UI占位，显示几条 Mock 消息即可，不需要真实通讯功能）。

3. **系统通知列表**：
   - 列表展示系统消息，如“系统通知：您的需求已被接单”、“系统通知：您有一笔报酬已到账”。
   - 未读通知前面加一个小红点。

4. **路由与导航**：
   - 路由设为 `/messages`。
   - 确保底部导航栏的“消息”按钮能正确跳转，且未读时图标上显示红点。
