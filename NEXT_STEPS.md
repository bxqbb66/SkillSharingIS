# 紧急任务：修复核心交互 Bug 及补充评价逻辑

暂停新功能开发，请优先修复以下 5 个严重影响用户体验和业务逻辑的问题：

1. **详情页缺少返回按钮**：
   - 问题：技能详情页、需求详情页、工单详情页点进去后，没有退回上一页的按钮。
   - 修复：在这三个详情页的顶部左上角，添加一个明显的“返回”按钮（左箭头 < 图标），点击后返回上一页。

2. **“我发布的”跳转逻辑错误**：
   - 问题：在个人中心，点击“我发布的技能”和“我发布的需求”，都跳转到了公共的首页广场。
   - 修复：新建两个独立页面展示当前用户发布的技能和需求；或者跳转回首页广场并自动带上“仅看自己”的筛选条件。请选择最合理的方案实现。

3. **消息聊天页面无内容**：
   - 问题：消息列表有摘要，但点进聊天界面是空的。
   - 修复：在聊天详情界面补充 Mock 数据，生成几条一问一答的对话记录。

4. **消息未读红点不消失**：
   - 问题：查看消息后，返回列表红点依然存在。
   - 修复：点击进入聊天界面时，将该消息的未读状态置为 false，消除红点。

5. **【业务逻辑缺失】评价完成后无处查看**：
   - 问题：工单完成后可以提交评价，但不知道去哪看评价结果。评价体系没有形成闭环。
   - 修复要求（必须做）：
     A. **实现“个人中心-我的评价”页面**：分为两个 Tab（我评价别人的 / 别人评价我的），展示星级和评价内容。
     B. **在“技能详情页”增加评价展示区**：在服务方信息附近或页面底部，展示该技能提供者收到的历史评价列表（头像、昵称、星级、评价文字），让别人了解他的口碑。
     C. **评价数据联动**：提交评价后，对应的工单状态应变为“已完成且已评价”，且在“我的评价”列表中能立刻看到这条新数据。
