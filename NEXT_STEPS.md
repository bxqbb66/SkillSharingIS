# 第六步开发任务：首页搜索筛选与登录注册闭环

核心交易闭环已完成，现在需要让功能真正可用。请完成以下两大模块：

## 一、首页搜索与分类筛选（高优先级）
目前首页的分类标签（全部、学业辅导等）和搜索框点击后没有反应，需要实现真实的筛选逻辑。

1. **分类标签筛选**：
   - 点击不同的分类标签（如“技术服务”），下方的卡片列表应自动过滤，只显示对应分类的技能/需求。
   - 切换“技能广场”和“需求广场”时，分类筛选条件应保持或重置。
   - 标签要有明显的选中态（如背景色变深、文字加粗）。

2. **搜索框功能**：
   - 在搜索框输入关键词（如“PPT”），点击搜索或回车后，卡片列表应根据“技能标签”或“需求标签”进行模糊匹配过滤。
   - 搜索结果为空时，展示友好的“暂无相关内容”提示页，而不是空白。

3. **组合逻辑**：
   - 搜索和分类可以叠加使用（比如：选了“学业辅导”分类，再搜索“高数”，应该能精准匹配）。

## 二、登录注册闭环（UI 及 Mock 逻辑）
目前顶部只有“登录 | 注册”文字，个人中心也是空白提示，需要补齐身份认证流程。

1. **登录/注册页面 UI**：
   - 点击顶部的“登录 | 注册”，或者个人中心的“请先登录”，跳转到一个统一的登录注册页面（路径：`/login`）。
   - 页面包含手机号/学号输入框、密码输入框。
   - 有“登录”和“去注册”的切换 Tab。
   - 注册页多一个确认密码和验证码输入框（UI占位即可，不需要真发短信）。

2. **Mock 登录逻辑（关键）**：
   - 使用 React 的 Context 或全局状态管理（如 Zustand/Redux，推荐最简单的 useState 传递）。
   - 点击“登录”按钮后，将状态标记为“已登录”，并将之前 Mock 的用户信息（张三，A 级信用等）存入状态中。
   - 登录成功后，自动跳转回首页。

3. **登录状态全局响应**：
   - **顶部导航栏**：未登录显示“登录 | 注册”；登录后显示“用户头像 + 姓名”。
   - **个人中心**：未登录显示“请先登录”占位；登录后显示之前做好的完整个人信息和工单列表。
   - **发布页面**：未登录点击发布，提示“请先登录”并跳转登录页；登录后正常填写并发布。

请确保这两个模块的逻辑连贯，特别是 Mock 的登录状态能准确在各个页面间同步。
