# 第二步开发任务：发布页面

请继续参考 CONTEXT.md 中的数据字典。首页已经开发完成，现在我们需要开发**“发布”功能**。

请创建两个页面组件：**发布技能页面** 和 **发布需求页面**。可以通过顶部的一个 Tab 来切换。

具体需求如下：

1. **整体布局**：
   - 手机端：全屏表单，底部有“提交发布”按钮。
   - 电脑端：表单居中展示，最大宽度 600px，两侧留白。

2. **发布技能表单**（严格对应 CONTEXT.md 中的 skill_information）：
   - 技能类别（skill_category）：下拉单选（学业辅导/技术服务/生活服务/文体服务）
   - 技能标签（skill_tag）：文本输入，提示“如：PPT制作、高数辅导”
   - 服务价格（service_price）：数字输入，单位元
   - 可服务时间（available_time）：文本输入，提示“如：周末全天、工作日晚8点后”
   - 技能描述（skill_description）：多行文本框
   - 资质信息（qualification_info）：文本输入（选填）
   - 附件上传：放一个“点击上传作品/证明”的按钮即可（UI占位，不需要真上传）

3. **发布需求表单**（严格对应 CONTEXT.md 中的 demand_information）：
   - 服务类型（service_type）：下拉单选（学业辅导/技术服务/生活服务/文体服务）
   - 需求标签（demand_tag）：文本输入，提示“如：求修电脑、求搬家帮忙”
   - 任务描述（task_description）：多行文本框
   - 预算金额（budget_amount）：数字输入，单位元
   - 截止时间：日期/时间选择器
   - 交付要求（delivery_requirement）：多行文本框（选填）
   - 紧急标记（urgent_flag）：开关按钮

4. **表单验证与交互**：
   - 必填项（类别、标签、描述、价格/预算）需要有红色的星号 * 标识。
   - 点击“提交发布”后，如果是空必填项，提示用户填写。
   - 提交成功后，弹出一个提示“发布成功，等待审核”，然后跳转回首页。

5. **路由配置**：
   - 将发布页面的路由添加到项目中，路径设为 `/publish`。
   - 确保首页底部导航栏中间的“发布（+）”按钮，点击后能跳转到这个发布页面。
