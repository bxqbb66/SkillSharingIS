## 🚀 新人如何跑起来？(3步搞定)

1. 确保你电脑装了 [Node.js](https://nodejs.org/) (下载 LTS 版) 和 Git。
2. 把仓库克隆到本地，并安装依赖包：
   bash
git clone <替换成你的仓库地址>
cd jlu-mutual-hub
npm install # 这一步必须等它跑完，千万别漏！

3. 启动项目：
bash
npm run dev

   终端会出现一个本地网址（如 `http://localhost:5173`），点开就能看到项目 ！

## 🚨 团队协作红线 (必看，违规请客奶茶！)

为了防止代码冲突导致大家白干，**严禁直接在主分支 `master/main` 上写代码！**

**正确的开发姿势：**
1. 每天开工前，先拉取最新代码：
bash
git checkout master
git pull

2. 新建你今天要写的功能分支（分支名用英文，见名知意）：
bash
git checkout -b feature/消息页面

或者修bug
git checkout -b fix/首页样式错乱

3. 在你自己的分支上写代码。
4. 写完后，推送到你自己的远程分支：
bash
git add .
git commit -m “feat: 完成了消息页面的UI”
git push origin feature/消息页面

5. **在 Gitee/GitHub 网页端发起 Pull Request (PR)**，通知队长来审核合并。**绝对不要自己合并到主分支！**

## 🤖 AI 协作开发指南 (如何正确使唤 Claude Code)

我们用 Claude Code 辅助开发，但 AI 没有记忆，它全靠读项目里的文件来理解上下文。项目里有三个灵魂文件：

- 📖 `CONTEXT.md`：**项目的基因库**。记录了所有 Mock 数据结构、页面路由和核心逻辑。**绝对不要删！** 如果 AI 改错了数据结构，去这里查。
- 📋 `NEXT_STEPS.md`：**任务看板**。你想让 AI 干什么活，先写在这里。
- 📝 `README.md`：就是你现在看的这个。

**和 AI 对话的正确模板：**
> 1. 先在 `NEXT_STEPS.md` 写好你的需求。
> 2. 在终端对 AI 说："我已经把需求写在 NEXT_STEPS.md 里了，请阅读并帮我开发。"

## 📦 Commit 提交规范

提交代码时，请按照以下前缀开头，方便大家看历史记录：
- `feat:` 新功能 (例: `feat: 新增钱包页面`)
- `fix:` 修Bug (例: `fix: 修复登录状态丢失`)
- `docs:` 改文档 (例: `docs: 更新README`)
- `style:` 改样式 (例: `style: 优化首页卡片圆角`)

## 👥 队长如何邀请协作者？

如果要让队友能推代码到仓库，必须把他们加为协作者：

### 🅰️ 如果用 Gitee (码云)
1. 打开仓库页面，点击右上角 **「管理」**。
2. 左侧菜单找 **「仓库成员管理」** -> **「添加仓库成员」**。
3. 输入队友的 Gitee 用户名或注册邮箱（需队友已注册）。
4. 权限选 **「开发者」**，点击添加。
5. 队友会收到站内信通知，接受后即可推代码。

### 🅱️ 如果用 GitHub
1. 打开仓库页面，点击 **「Settings」**。
2. 左侧菜单点击 **「Collaborators」**。
3. 点击 **「Add people」**，输入队友的 GitHub 用户名或邮箱。
4. 权限选 **「Write」**，点击邀请。
5. ⚠️ **关键**：队友必须去邮箱点击确认链接接受邀请，才能推代码！（经常有人漏掉这步）
